# Houzs ERP ‚Äî design-sync notes

## Shape
- `package` shape on an **application** repo (not a library) ‚Äî synth-entry mode from `src/components/`.
- 48 components in scope (`src/components/*.tsx` + `src/components/scm-v2/*.tsx`).
- No `.d.ts` declarations shipped ‚Äî types inferred from `.tsx` via ts-morph.

## Tailwind CSS pipeline
- Theme C tokens live in `tailwind.config.js` (not a tokens package).
- `cssEntry` points at `.design-sync/tailwind-built.css`, regenerated by `cfg.buildCmd`.
- The Tailwind build is **scoped** to `./src/components/**/*.tsx` so the bundled CSS only carries classes the components actually use.

## Provider context (not yet wired)
The app's real provider chain is:
```
GlobalSearchProvider > NotificationsProvider > BreadcrumbsProvider
  > Layout > BrowserRouter > useAuth() (AuthContext)
```
Many components also call `useQuery` from `../hooks/useQuery` (custom hook, not `@tanstack/react-query`).

Starting without `cfg.provider` ‚Äî will add as `[RENDER]` errors surface specific contexts.

## Re-sync risks
- `buildCmd` rebuilds the Tailwind CSS but **does NOT** rebuild the app; a new component added to `src/components/` will be picked up by synth-entry, but its utility classes must be present in tailwind-built.css (the buildCmd re-scans content globs so this should hold).
- Tailwind v3 syntax in `tailwind.config.js` ‚Äî if upgraded to v4 the build command changes.
- This is an internal app, not a published package ‚Äî there is no `dist/index.js` to bundle from. Synth-entry mode produces weaker `.d.ts` for components; `cfg.dtsPropsFor` accumulates fixes.

## Token packages
- Vendored 2990 SCM tokens at `src/vendor/design-system/tokens.css` ‚Äî pulled in via `tokensGlob`. The `:root` block was copied verbatim from 2990's design-system package and remapped to Theme C colors (cream‚Üíwhite, beige‚ÜíTheme C surface-dim).

## Fonts
- IBM Plex Mono (used only via `font-money` for amounts) is **not shipped**; suppressed via `runtimeFontPrefixes: ["IBM Plex"]`. Money cells fall back to `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace` ‚Äî acceptable substitute per owner.
